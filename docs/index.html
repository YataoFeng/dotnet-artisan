Core Principles

KISS First

Simple CRUD does not need DDD or CQRS. Match the pattern to the problem scale.

No Repository Wrappers

Use DbContext directly. EF Core IS your repository and unit of work.

Version Adaptive

Auto-adapts from net8.0 through net11.0. Knows what's available in each TFM.

Free & Open Source

Microsoft-recommended + MIT-licensed packages only. Zero commercial dependencies.

Most common mistakes vs correct patterns. Click to expand.

DI Lifetime Mismatch Critical▼

Bad — Scoped injected into Singleton
builder.Services.AddSingleton<ICache, MyCache>();
public class MyCache(AppDbContext db) : ICache { } // DbContext is Scoped!
Good — Use IServiceScopeFactory in Singleton
public class MyCache(IServiceScopeFactory sf) : ICache
{
  public async Task<T> Get<T>(string k, Func<AppDbContext,Task<T>> f) {
    using var scope = sf.CreateScope();
    return await f(scope.ServiceProvider.GetRequiredService<AppDbContext>());
  }
}

DateTime.Now → TimeProvider Common▼

Bad — Untestable
var order = new Order { CreatedAt = DateTime.Now };
Good — Injectable, testable
public class OrderService(AppDbContext db, TimeProvider clock) {
  var order = new Order { CreatedAt = clock.GetLocalNow().DateTime };
}

EF Core N+1 Queries Perf▼

Bad — One query per child
var orders = await db.Orders.Where(...).ToListAsync();
foreach (var o in orders)
  o.Items = await db.OrderItems.Where(i => i.OrderId == o.Id).ToListAsync();
Good — Single Include query
var orders = await db.Orders.Include(o => o.Items).AsNoTracking().ToListAsync();

Repository Wrapping DbContext Anti-pattern▼

Bad — Useless abstraction, hides IQueryable
public interface IRepository<T> { Task<T> GetById(int id); }
public class Repository<T>(AppDbContext db) : IRepository<T> { }
Good — Use DbContext directly
public class OrderService(AppDbContext db) {
  public async Task<Order?> Get(int id) => await db.Orders.FindAsync(id);
}

.Result / .Wait() Deadlock Critical▼

Bad — Blocks thread, deadlock risk
var data = GetDataAsync().Result; // Sync-over-async = deadlock
Good — Async all the way
var data = await GetDataAsync();

Missing CancellationToken Common▼

Bad — Uncancelable
public async Task<List<Product>> Search(string q)
  => await db.Products.Where(p => p.Name.Contains(q)).ToListAsync();
Good — Propagate CancellationToken
public async Task<List<Product>> Search(string q, CancellationToken ct = default)
  => await db.Products.Where(p => p.Name.Contains(q)).ToListAsync(ct);

new HttpClient() Socket Exhaustion Perf▼

Bad — New connection per request, port exhaustion at scale
using var client = new HttpClient();
var result = await client.GetStringAsync(url);
Good — IHttpClientFactory manages connection pool
public class ApiClient(IHttpClientFactory factory) {
  public async Task<string> Get(string url) {
    var client = factory.CreateClient("api");
    return await client.GetStringAsync(url);
  }
}

Exceptions for Control Flow Critical▼

Bad — Exception for expected business outcome
public Order GetOrder(Guid id) {
  return db.Orders.Find(id) ?? throw new NotFoundException();
}
Good — Result pattern for expected failures
public Result<Order> GetOrder(Guid id) {
  var order = db.Orders.Find(id);
  return order is not null ? Result.Success(order) : Result.Failure();
}

Hardcoded Secrets Security▼

Bad — Secrets committed to source control
const string ApiKey = "sk-abc123xyz";
var connStr = "Server=prod;Password=Secret123!";
Good — User Secrets + env vars / Key Vault
var apiKey = builder.Configuration["ApiKey"];
var connStr = builder.Configuration.GetConnectionString("Default");

Swashbuckle on .NET 10 Obsolete▼

Bad — Swashbuckle is unmaintained, no Native AOT support
builder.Services.AddSwaggerGen();
app.UseSwagger(); app.UseSwaggerUI();
Good — Built-in OpenAPI + Scalar
builder.Services.AddOpenApi();
app.MapOpenApi(); // /openapi/v1.json
app.MapScalarApiReference(); // Modern API docs UI

Non-Negotiable Rules

1. No Repository / UoW

DbContext IS the unit of work, DbSet<T> IS the repository. Do not wrap them in meaningless IRepository and IUnitOfWork interfaces.

2. No FluentValidation

.NET 10+ use built-in AddValidation() + DataAnnotations. Source-generated, AOT-compatible, zero dependencies.

3. No DateTime.Now

Use TimeProvider everywhere. Constructor-inject it. Unit test with TimeProvider.Fake().

4. Free / Open Source Only

MediatR→Mediator(MIT), AutoMapper→Mapperly, Newtonsoft→STJ, Swashbuckle→Microsoft.AspNetCore.OpenApi.

5. DbContext Directly

Inject AppDbContext into handlers. Do not create IOrderRepository. EF Core is already a complete repository implementation.

6. Async All the Way

No .Result, .Wait(), or async void (except UI events). CancellationToken end-to-end propagation.

7. Parameterized Queries

No SQL string concatenation. Use FromSqlInterpolated or LINQ. Secrets in user-secrets / Key Vault, never hardcoded.

8. Testcontainers for Integration Tests

Real databases via Testcontainers. No InMemory. Test behavior, not implementation details.
