核心原则

KISS 优先

简单 CRUD 不需要 DDD 或 CQRS。按问题规模选择模式。

禁止仓储包装

直接用 DbContext。EF Core 本身就是仓储和工作单元。

版本自适应

自动适配 net8.0 到 net11.0。

免费 & 开源

仅用微软推荐 + MIT 协议包。零商业依赖。

最常见的错误 vs 正确写法。点击展开查看详情。

DI 生命周期不匹配 严重▼

错误 — Scoped 注入到 Singleton
builder.Services.AddSingleton<ICache, MyCache>();
public class MyCache(AppDbContext db) : ICache { } // DbContext 是 Scoped！
正确 — 在 Singleton 中用 IServiceScopeFactory
public class MyCache(IServiceScopeFactory sf) : ICache
{
  public async Task<T> Get<T>(string k, Func<AppDbContext,Task<T>> f) {
    using var scope = sf.CreateScope();
    return await f(scope.ServiceProvider.GetRequiredService<AppDbContext>());
  }
}

DateTime.Now → TimeProvider 常见▼

错误 — 不可测试
var order = new Order { CreatedAt = DateTime.Now };
正确 — 可注入可测试
public class OrderService(AppDbContext db, TimeProvider clock) {
  var order = new Order { CreatedAt = clock.GetLocalNow().DateTime };
}

EF Core N+1 查询 性能▼

错误 — 每个订单一次查询
var orders = await db.Orders.Where(...).ToListAsync();
foreach (var o in orders)
  o.Items = await db.OrderItems.Where(i => i.OrderId == o.Id).ToListAsync();
正确 — 一次 Include 查询
var orders = await db.Orders.Include(o => o.Items).AsNoTracking().ToListAsync();

仓储包装 DbContext 反模式▼

错误 — 无用抽象，隐藏 IQueryable
public interface IRepository<T> { Task<T> GetById(int id); }
public class Repository<T>(AppDbContext db) : IRepository<T> { }
正确 — 直接用 DbContext
public class OrderService(AppDbContext db) {
  public async Task<Order?> Get(int id) => await db.Orders.FindAsync(id);
}

.Result / .Wait() 死锁 严重▼

错误 — 同步上下文死锁风险
var data = GetDataAsync().Result; // 阻塞线程，可能死锁
正确 — 异步一路到底
var data = await GetDataAsync();

缺少 CancellationToken 常见▼

错误 — 无法取消
public async Task<List<Product>> Search(string q)
  => await db.Products.Where(p => p.Name.Contains(q)).ToListAsync();
正确 — 传播 CancellationToken
public async Task<List<Product>> Search(string q, CancellationToken ct = default)
  => await db.Products.Where(p => p.Name.Contains(q)).ToListAsync(ct);

不可协商的铁律

禁止 Repository/UoW

DbContext 就是工作单元。DbSet<T> 就是仓储。

禁止 FluentValidation

.NET 10+ 用 AddValidation() + DataAnnotations。

禁止商业包

MediatR→Mediator(MIT)、AutoMapper→Mapperly、Newtonsoft→STJ。

禁止 DateTime.Now

全项目使用 TimeProvider。可注入，可测试。

直接注入 DbContext

注入 AppDbContext，禁止 IRepository<T>。

仅免费/开源

详见 package-choices.md 全部替代方案。
